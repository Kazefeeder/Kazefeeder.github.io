hexo博客搭建全流程记录（持续更新中）

发表于 2019-11-29 分类于 写作 ， 编程
自己搭建的笔记

准备工作

  • 下载node.js并安装，默认会安装npm。(官网下载安装）
  • 下载安装git（官网下载安装）
  • 下载安装hexo。方法：打开cmd 运行npm install -g hexo

搭建本地的静态博客

网站模板生成

新建一个文件夹，如hexo，进入这个文件夹后，右键打开“git Bash Here”，打开git，输入 hexo init，即可生成网站模板。
最后运行：hexo server （可跳过）（运行程序，访问本地localhost:4000可以看到博客已经搭建成功，）

主题安装

访问主题列表，选择自己喜欢的主题，点进去按照主题的之时安装。
一般的命令为：

打开站点__config.yml文件，将themes修改下载到的主题文件夹的名字

  • hexo g
  • hexo s(本地端口看效果）
  • hexo d(推送到github上全网展示)

更新博客内容

更新文章
  1. 在MyBlog目录下执行：hexo new “我的第一篇文章”，会在source_posts文件夹内生成一个.md文件。
  2. 编辑该文件（遵循Markdown规则）
  3. 修改起始字段

    起始字段就是在md文件顶上自动生成的—中间的部分，官网上英文的称呼叫做 front-matter，这里面包括日期，标签，分类，密码，摘要等内容，用的时候分类、标签等功能要先添加，否则会出现网站渲染失败的情况。

  1. 编写正文内容（MakeDown）
  2. hexo clean 删除本地静态文件（Public目录），可不执行。
  3. hexo g 生成本地静态文件（Public目录）
  4. hexo deploy 将本地静态文件推送至github（hexo d）

向网络上部署博客

将博客与Github关联

  • 在Github上创建名字为XXX.github.io的项目，XXX为自己的github用户名。

  • 打开本地的MyBlog文件夹项目内的_config.yml配置文件，将其中的type设置为git

    记得替换上面的yourname,而且冒号后面有空格
    本文中所有的yourname都是指你的github账号id

  • 运行：npm install hexo-deployer-git –save

  • 运行：hexo g（本地生成静态文件）

  • 运行：hexo d（将本地静态文件推送至Github）
    此时，打开浏览器，访问http://yourname.github.io

    绑定域名

博客已经搭建好，也能通过github的域名访问，但总归还是用自己的域名比较舒服。因为我们需要设置将自己的域名绑定到github这个博客项目上。
域名提供商设置添加2条A记录：

博客添加CNAME文件 配置完域名解析后，进入博客目录，在source目录下新建CNAME文件，写入域名，如：kazefeeder.cn
进入项目仓库—settings，向下找到Github Pages，将 Custom domain 添入自己的域名并保存

  • 运行：hexo g
  • 运行：hexo d(也可以直接: hexo d -g)
